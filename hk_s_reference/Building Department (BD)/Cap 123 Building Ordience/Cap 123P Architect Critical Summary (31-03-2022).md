# Cap 123P — Building (Inspection and Repair) Regulation
**Architect critical summary for schematic design**  
Consolidated version: 31 March 2022 | Cap. 123 sub. leg. P

> Scope note: Cap 123P is the **MBIS / MWIS inspection-scope, repair-standard, and RI / QP document rulebook** under Cap 123 ss.30B–30E. It is **not** plan approval (Cap 123A), plot-ratio control (Cap 123F), construction quality (Cap 123Q), or the notice/age triggers themselves (those live in Cap 123). Use this before schematic design on any **existing building** A&A, façade works, common-parts upgrade, projection / appendage design, or window replacement where a prescribed inspection / repair may already apply — or will govern maintainability of what you design.

---

## Regulatory Overview

Cap 123P defines what counts as a **prescribed inspection** and **prescribed repair**, what building elements must be examined (common parts, external walls, prescribed projections, signboards, windows), and to which construction standards inspection and repair are measured. It also locks the Registered Inspector (RI) / Qualified Person (QP) appointment, report, repair-proposal, independence, and handover duties that run alongside Cap 123’s Mandatory Building Inspection Scheme (MBIS) and Mandatory Window Inspection Scheme (MWIS).

## Critical main topics and subtopics

### 1. When Cap 123P bites at schematic design

| Project situation | Why Cap 123P matters at SD |
|---|---|
| Existing building ≥ notice age / already under Cap 123 s.30B–30C notice | Inspection scope (Schedules 1–2 + projections + signboards + windows) becomes a **condition survey / repair budget** driver before you freeze massing or façade package |
| A&A to common parts, lobby, MOE, fire-resisting construction, drains | Those items are **prescribed inspection items** (Sch. 1) — defects found trigger prescribed repair to “rendered safe” |
| Façade / cladding / curtain wall / louvres / fins | External-wall items (Sch. 2) + finishes (tiling, rendering, cladding, false ceiling) are in scope |
| Balconies, verandahs, planter boxes, drying racks, canopies, BS supports / pipes | Prescribed as **projections** (s.9) — inspectable and repairable under the Ordinance’s projection limb |
| Window replacement, aluminium window A&A, MWIS response | Separate **QP** track for windows (s.4(1)(e), s.14) — windows excluded from “common parts” and “external wall” limbs (s.4(2)–(3)) |
| Signboard on building | Display surfaces, fixings, and supporting structures are inspectable (s.4(1)(d)) |
| New-build only on cleared site | Cap 123P generally **does not drive** new-build SD; design still for future inspectability if the asset will later fall under MBIS/MWIS |

> Cap 123P does **not** itself set the building-age / notice thresholds. Confirm whether Cap 123 ss.30B–30E notices apply (or are imminent) before treating this Cap as dormant.

### 2. What a prescribed inspection must cover (s.4) — the SD survey checklist

An examination/assessment is a prescribed inspection only if it covers **all applicable items** below (including **finish** to each item):

| Limb | Must cover | Architect note |
|---|---|---|
| **(a) Common parts** | All applicable Sch. 1 **and** Sch. 2 items (components + finish) | Sch. 2 items sit inside common-parts inspection when they are part of common parts |
| **(b) External wall** | All applicable Sch. 2 items (components + finish) | External-wall limb **excludes windows** on that wall (s.4(3)) |
| **(c) Projection** | Components of any projection prescribed in **s.9** (incl. finish) | See §4 — design appendages as projection types from day one |
| **(d) Signboard** | Display surfaces, fixings, supporting structures | Fixings / structure often control façade interface and load path |
| **(e) Window** | Components of the window | Own QP regime; not bundled into common-parts / EW limbs (s.4(2)–(3)) |

**Finish (s.4(4))** includes any **tiling, rendering, cladding, or false ceiling**.  
**SD takeaway:** Specifying cladding / stone / tile / false ceiling on external wall or common parts places those finishes squarely inside future MBIS scope and repair liability.

### 3. Schedule 1 & 2 inspection items — design elements that will be audited

#### 3.1 Schedule 1 — Common parts (other than external walls)

| Item | What RI must be able to examine | SD / A&A implication |
|---|---|---|
| **1. Building structures** | Transfer structures, cantilevered structures, **exposed pile caps** | Do not hide transfer / cantilever / exposed pile-cap zones behind uninspectable finishes without access strategy |
| **2. Fire safety provisions** | Means of escape; means of access for firefighting & rescue; **fire-resisting construction** | Lobby / stair / FRR package is an MBIS item — A&A that strips FRR or MOE geometry creates inspection/repair findings |
| **3. Common drains** | Aboveground & underground common drain pipes + associated fixings | Riser / duct / underground drain routes need access for inspection and repair |
| **4. Fixtures & installations** | Metal gates, fence walls, protective barriers, parapets, balustrades, skylights | Guarding, roof lights, and courtyard gates are in scope — not “soft FF&E” |

#### 3.2 Schedule 2 — External walls

| Item | What RI must be able to examine | SD / A&A implication |
|---|---|---|
| **1. Building structures** | Transfer structures, cantilevered structures | Bay windows, canopies tied into structure, EW transfers — structural exposure / access |
| **2. Non-structural items** | Fins, grilles, metal louvers | Architectural screens and acoustic / visual fins are inspectable fabric |
| **3. Curtain walls** | Curtain wall systems | CW package = MBIS external-wall item; design anchors, access, and replaceability |
| **4. Appendages / projections / fixtures** | Architectural projections; fixtures incl. metal gates, protective barriers, parapets, balustrades | Façade ornament, edge protection, and gates are core inspectable set |

**SD takeaway:** Anything you put on the EW as “architecture” is likely an inspection target. Budget access (gondola / BMU / reachable terraces) and demountable detailing for future prescribed repair.

### 4. Prescribed projections (s.9) — hard list for façade / A&A

For Cap 123 s.30B(5), each of the following **is a projection**:

| Projection type | Covered? | Notes |
|---|---|---|
| Balcony | Yes | |
| Verandah | Yes | |
| Planter box | Yes | |
| Drying rack | Yes | |
| Window canopy | Yes — only if it meets s.9(2) | See below |
| Supporting structure for a **building service installation** (excl. drainage) | Yes | AC platforms, cable trays supports, BS plant supports on façade |
| Pipe or duct associated with that BS installation | Yes | Condensate / refrigerant / duct runs on EW |

**Window canopy test (s.9(2)) — all three must be true:**
1. Projects from an external wall and is **located above an opening**; **and**
2. **Does not carry any floor load**; **and**
3. Is **cantilevered or supported by brackets**.

**SD takeaway:** If a “canopy” carries floor load or is not over an opening, it may fall outside this prescribed projection definition — but it may still be Sch. 2 appendage / structure. Classify early so the RI / AP pathway is correct.

### 5. Inspection & repair standards (ss.5 & 8) — do not upgrade-by-stealth unless intended

#### 5.1 Inspection purpose (s.5(1))
Prescribed inspection must, by reference to the standards in s.5(2):
1. Ascertain whether the building is **safe**, **liable to become dangerous**, or **has been rendered dangerous**;
2. Identify **defects / deficiencies**; and
3. Make a **proposal for any prescribed repair**.

#### 5.2 Benchmark standards (s.5(2)) — use the **latest** applicable (s.5(3))

| Standard source | Meaning at SD |
|---|---|
| **(a)** Works as shown on **BA-approved plans** to which the building was completed | Default: original approved construction |
| **(b)** Plans submitted under **Cap 123N** (Minor Works) | Later MW that legally altered the fabric sets a newer benchmark |
| **(c)** Improvement works completed under **Cap 502** or **Cap 572** (fire safety improvement) | FS upgrades completed under those Ordinances raise the comparison bar |
| **(d)** Standard of design & construction **prevalent at time of construction** | Gap-filler when plans are incomplete — era code, not today’s code |

#### 5.3 Prescribed repair outcome (s.8)
| Action | Required effect |
|---|---|
| **Repair** | Building / element must be **rendered safe** after repair (s.8(1)) |
| **Testing** | Verify inspection findings; or prove building is safe / has been rendered safe (s.8(2)) |
| Benchmark | Same s.5(2) standards; if multiple apply → **latest** (s.8(3)–(4)) |

**SD takeaway:** Prescribed repair is **restore-to-safe against the latest applicable historical standard**, not an automatic full upgrade to current Cap 123F / FS Code. If the scheme intends code upgrades beyond prescribed repair, that is a parallel A&A / OP amendment path — do not conflate the two in cost or programme.

### 6. Team split: RI vs QP vs contractor (design programme roles)

| Role | Cap 123 hook | Cap 123P duty that affects SD programme |
|---|---|---|
| **Registered Inspector (RI)** | Cap 123 s.30D | Building (non-window) prescribed inspection & supervision of prescribed repair |
| **Qualified Person (QP)** | Cap 123 s.30E | Window prescribed inspection & repair supervision |
| **QP representative** | Cap 123 s.30E(4) | If QP is RGBC or RMWC → **authorized signatory** is the prescribed representative (s.10) |
| **Registered contractor** | Cap 123 | Executes prescribed repair; must keep inspection report / proposal on site (s.32) |

#### 6.1 Appointment clocks (s.11)
RI / QP must notify BA of appointment in specified form **within 7 days**, signed by the RI/QP **and** the person for whom the works are carried out.

#### 6.2 Independence — RI must not own the contractor (ss.12(3), 13(2)(c))
Where repair is required and (as applicable) on completion of repair, RI certifies they are **not** a partner, director, or authorized signatory of the registered contractor doing the repair.  
**SD takeaway:** Do not propose a single company as both inspecting RI-side entity and repairing contractor for building (non-window) prescribed repair without checking independence.

### 7. Document packages & hold points (ss.12–14) — programme gates

#### 7.1 Building inspection complete (RI under s.30D(1)(a)) — **within 7 days** (s.12; windows excluded)
Submit to BA:
1. **Inspection report** — methods; findings (records/tests); assessment; (if applicable) **repair proposal** to render safe; (if applicable) report of building works that Cap 123 s.30D(5)(b) requires the RI to notify BA of; **and**
2. **Certificate** — inspection done per Ordinance; building is **safe / no repair**, **or** dangerous / liable to become dangerous **and repair required**; report prepared per Ordinance.

If repair required and a **different** RI is appointed under s.30D(1)(b): inspecting RI also files an independence certificate **within 14 days after repair completion** (s.12(3)).

#### 7.2 Building repair complete (RI under s.30D(1)(b)) — **within 14 days** (s.13)
Submit:
1. **Completion report** — repair records; material certificates; testing methods & results; account of **all revisions** to the original proposal;
2. **Certificate** — repair per Ordinance; per original or **revised** proposal (s.22); building **rendered safe**; report compliant;
3. **Independence certificate** — RI not partner/director/AS of the contractor.

#### 7.3 Windows (QP) — condensed rules (s.14)
| Scenario | Deadline | Core deliverable |
|---|---|---|
| Inspect only, **no repair** (same QP track s.30E(1)(a)) | **14 days** after inspection | Certificate: inspected per Ordinance; window **safe** |
| Same QP does inspection **and** repair (s.30E(1)(a)+(b)) | **14 days** after repair | Certificate: was dangerous/liable; repair done; window **rendered safe** |
| Different QPs for inspect vs repair | Inspect QP: **7 days** after inspection | Inspection report (findings, assessment, repair proposal) + “repair required” certificate |
| | Repair QP: **14 days** after repair | Certificate: repair per Ordinance & proposal / revised proposal; window rendered safe |

### 8. Detailed investigation trigger (ss.15–21) — when schematic cannot proceed on walkover alone

Applies to **building** prescribed inspection — **not** windows (s.15).

#### 8.1 When detailed investigation may be run (s.17)
During prescribed inspection, if RI finds:
- a **serious defect** constituting **structural instability** or **serious health hazard**; **or**
- the **extent or cause** of a defect **cannot be ascertained**,

then, to make a prescribed-repair proposal, a detailed examination/assessment of any Sch. 1 / Sch. 2 item, projection, or signboard may be conducted.

#### 8.2 BA endorsement path (ss.18–20) — design / programme critical
| Step | Rule |
|---|---|
| Before start | RI must **notify BA in writing** and submit a **detailed-investigation proposal** for endorsement (s.18) |
| Proposal contents | Purpose; scope/method/particulars; method justification; summary of defects + annotated photos + marked-up plans |
| BA decision | Endorse in whole/part or refuse — BA must decide **within 28 days** of receiving notice + proposal (s.19) |
| Default start rule | **Do not commence** detailed investigation until BA endorsement received (s.20(1)) |
| Client override | May proceed **without** endorsement only if client is told of BA’s refusal and **decides to proceed anyway** (s.20(2)) |
| Specialist | RI may engage & supervise a specialist — this is an exemption for Cap 123 s.30D(3)(a) (s.21) |

**SD takeaway:** Serious structure / health findings or unknown defect causation add a **BA endorsement + up to 28-day decision** hold before repair scheme (and often before A&A that depends on that fabric) is reliable.

---

### 9. Repair must follow the proposal — revise within 7 days (ss.22–23)

| Who | Duty |
|---|---|
| RI supervising building repair (s.30D(1)(b)) | Ensure repair is carried out **per the s.12 repair proposal**; if circumstances require change → submit **revised proposal to BA within 7 days** (s.22) |
| QP supervising window repair only (s.30E(1)(b) but not (a)) | Same logic vs s.14(4) proposal; revised proposal within **7 days** (s.23) |

**SD takeaway:** Prescribed repair is proposal-controlled work. Unapproved “while we’re up there” A&A extras on the same scaffold still need their own legal pathway; do not let them silently morph the prescribed-repair proposal without an s.22 / s.23 revision.

---

### 10. Supervision continuity & stop-work rules (ss.24–29)

| Rule | Practical effect |
|---|---|
| RI for repair may appoint a **representative** (s.24) | Notify BA **≥ 7 days before** appointment with particulars/qualifications/experience; RI remains **personally responsible** |
| BA may **reject** representative for insufficient/unsuitable quals/experience (s.25) | Appointment ceases on BA’s stated rejection date |
| RI / QP ceases to act (s.27) | Notify BA in writing within **7 days** of cessation |
| Temporary nominee under Cap 123 s.30D(10) (s.28) | Notify BA within **7 days** of nomination; notify again within **7 days** if nomination ceases |
| **Stop repair** if supervising RI/QP unwilling/unable and no valid replacement notified (s.29) | Building repair stops until new RI under s.30D(1)(b) + s.11 notification; window repair stops until new QP under s.30E(1)(b) + s.11 |

**SD takeaway:** For long façade / common-parts repair phases tied to an A&A scheme, continuous RI/QP cover is a **programme risk**; plan deputies under s.30D(10) / representative rules early.

### 11. Information flow duties that protect the design team (ss.30–32)

| Duty | Clock / rule |
|---|---|
| Inspecting RI → client copies of s.12(2) pack | Within **7 days** after inspection completion |
| Inspecting RI → repairing RI (if different) | Within **2 months** of BA submission **or** before repair starts — **whichever earlier** |
| Repairing RI → contractor | Inspection pack **before** repair starts; revised proposal **same day** as BA submission |
| Repairing RI → client | Completion / revised docs **same day** as BA submission |
| QP window tracks | Parallel mirror rules with **1-month** max for inspect-QP → repair-QP handover (s.30(3)–(5)) |
| BA additional information | RI / QP must supply anything BA requires on submitted packs (s.31) |
| Contractor on site | Must keep on site the inspection report / proposal (and any revised proposal) and produce to BA on demand (s.32) |

### 12. Document quality bar (ss.33–36) — submission hygiene

| Rule | Requirement |
|---|---|
| Completion (s.33) | Clear, correct, duly signed |
| Material / form (s.34) | Clear & intelligible; if **hard copy**, on suitable durable material (hard copy = paper or similar readable without equipment) |
| Authorship / signature (s.35) | Inspection reports/proposals prepared by/under RI or QP appointed for inspection; repair reports/revised proposals by/under RI or QP appointed for repair; signer **assumes all Ordinance responsibilities** for that document |
| Non-derogation (s.36) | Cap 123P duties do **not** dilute any other Cap 123 or enactment duty |

### 13. Schematic-design go / no-go gate (architect checklist)

Before freezing existing-building A&A, façade, common-parts, projection, or window packages:

1. **Confirm Cap 123 ss.30B–30E status** — active notice, impending cycle, or none. Cap 123P only operationalises prescribed inspection/repair once that framework applies.
2. **Map fabric to limbs** — common parts (Sch. 1 + applicable Sch. 2), external walls (Sch. 2), s.9 projections, signboards, windows (separate QP track).
3. **Classify every appendage** — balcony / verandah / planter / drying rack / window canopy (s.9(2) test) / BS support & associated pipes-ducts vs Sch. 2 appendage / structure.
4. **Finish schedule risk** — tiling / rendering / cladding / false ceiling on EW or common parts = inspectable finish (s.4(4)).
5. **Access & demountability** — transfer / cantilever / exposed pile caps / CW / fins / louvres / guarding must be inspectable and repairable without destroying your design.
6. **FRR / MOE / firefighting access** — treat as MBIS Sch. 1 items; do not propose A&A that creates obvious deficiency against the **latest applicable** s.5(2) standard unless a parallel approved upgrade path is funded.
7. **Standard baseline** — pull approved plans, Cap 123N records, Cap 502 / 572 FS improvement records; if conflict, latest wins (s.5(3)).
8. **Repair vs upgrade** — prescribed repair = **rendered safe** to historical latest standard; full current-code upgrade is separate A&A.
9. **Team independence & clocks** — RI ≠ contractor interest; appointment / report / repair / revised-proposal deadlines are short (7–14 days typical).
10. **Unknown structure / serious defect** — budget detailed investigation + BA endorsement (up to 28 days) before locking structural A&A that depends on that fabric.
11. **Site documentation** — contractor must hold inspection/proposal set on site; design changes mid-repair need s.22 / s.23 revised proposals.
12. **Cross-link** — Cap 123 (ss.30B–30E duties & notices), Cap 123A (if parallel building works / plans), Cap 123N (minor works that reset the s.5 benchmark), Cap 502 / 572 (FS improvement standards).
